该个人博客使用GitHub+Hexo搭建，点击文档了解更多关于hexo的使用信息，主题选用为Beantech

配置环境需求

Node.js
Git
Hexo
上述需求均可在各自官网下载安装

Github相关设置

Node.js和Git都安装完成后，我们需要申请一个GitHub账号，以供之后blog的部署
账号申请完成后，点击头像左边的 “+” 符号，创建一个新的 “仓库”

填入 “仓库” 名称，需为”username.github.io“，若为其他名称，个人主页或许会失效，图中因为博主已创建过个人主题，所以重名报错，没有影响

仓库创建完成后需要配置SSH KEY，用以本地向GitHub服务器进行文件传输
使用下述命令配置Git的用户信息

生成SSH秘钥文件

秘钥文件的默认生成位置为C:\user\yourname\.ssh文件夹下

生成两个文件，“.pub”为公钥，需要将里面的内容复制放入GitHub账号“设置”中，另一个文件为私钥文件，需要妥善保存，不要泄露
注：若需要在不同的电脑上进行管理，需要在各个电脑上生成SSH KEY，并将它们添加到GitHub账号中

Hexo相关设置

  1. 运行npm install -g hexo安装hexo本地环境
  2. 创建保存本地博客的工作文件夹MyBlog
  3. 初始化hexo博客工程：hexo init
  4. 编译博客系统：hexo g
  5. 启动本地服务器进行预览：hexo s

如果hexo正常工作了，输入 “http://localhost:4000/” 就能看到博客的初始模样

我们在GitHub上部署，还需要安装插件hexo-deployer-git

更多信息请点击文档进行了解

发布新的文章

在选定自己喜欢的主题后，使用“hexo new yourpapertitle”进行新的文章创建，在“source/_posts”文件夹下回生成“yourpapertitle.md”文件，这就是你的文章

部署文章

要想将写好的文章挂在GitHub服务器上，需要我们对GitHub进行关联，这就是之前获取的SSH KEY的作用
此外，我们还需要在blog根目录下的配置文件中“_config.yml”进行数据的补充（注：根目录下的_config.yml,不是themes文件夹下的_config.yml）

在“：”后需要加一个空格，否则配置无效
文章编辑号后，即可生成、部署文章

这样一篇blog就完成了，可喜可贺可喜可贺！！
